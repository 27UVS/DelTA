## [Unreleased]

### Fixed

- **Истории (роли / исполнители):** при редактировании истории не все отмеченные исполнители сохранялись. Исправлено: при пересборке чипов (в том числе из‑за поиска по роли) выделение берётся из черновика, а не с диска; программная установка «нажато» на чипах не шлёт ложные `toggled`; при каждом клике список выбранных для роли обновляется от актуального `_draft`, а не от снимка на момент последней пересборки (из‑за этого второй и следующие выборы в одной роли затирали предыдущие).
- **Истории:** добавлена кнопка **«Полное удаление»** — удаляет историю из базы (не архив) и автоматически снимает связи с задачами (поле `story_id` очищается).
- **Карточки задач:** если задача привязана к истории — показывается строка **«Привязанная история: <название>»** с многострочным переносом.
- **Карточки задач:** при переименовании истории название в строке «Привязанная история» обновляется (перестройка карточек при возвращении на доску).

### Changed

- **Статусы историй:** обновлены цвета под тёмную тему; статус **«Опубликован»** сделан более ярким бирюзовым.
- **Настройки интерфейса:** убран выбор темы — приложение использует только тёмную тему.

## [v1.5.0] - 2026-04-26

### Added

- **Режим «Истории» (планирование сценария):** отдельная страница с фильтрами (сезон, арка, разделы, архив, поиск), списком историй и панелью деталей; переключение с доски кнопками «дом» / «буфер».
- **Данные историй:** `stories.json`, `story_statuses.json`, `story_taxonomy.json`; статусы историй с цветами; сезоны / арки / разделы с защищёнными системными записями («Все», «Актуальные» / «Не актуальные»).
- **Админка → Таблицы:** вкладка **«Сюжет»** для управления сезонами, арками и разделами.
- **Роли для сценария:** у роли флаг `for_stories` (чекбокс в диалоге роли); на странице «Истории» блоки назначения людей строятся только по ролям с этим флагом.
- **Редактирование истории:** явные кнопки «Редактировать» / «Сохранить» / «Отмена» и черновик без автосохранения при смене фильтров или выбора другой истории.
- **Карточки историй в списке:** крупное многострочное название (цвет по статусу), статус, справа — исполнители цветами ролей; высота карточки учитывает переносы и число исполнителей.
- **Порядок списка историй:** режимы **Ранние** / **Поздние** / **Персональная** (по умолчанию персональная); при возврате с сортировки по дате персональный порядок подхватывает видимый порядок; в персональном режиме — перетаскивание карточек внутри списка с сохранением поля `order`.
- **Связь задачи с историей** (только при **экспериментальном режиме**): в создании/редактировании задачи блок **«Связанная история»** — один выбор из списка с поиском по названию; в просмотре задачи — переход на страницу «Истории» с автовыбором связанной истории.
- **Экспериментальный режим** в профиле администратора: переключатель под «Другие ссылки/контакты»; при выключении скрываются «Истории» и кнопки переключения; данные историй в `db/` сохраняются.
- **Панель «Люди»:** на странице «Историй» счётчик показывает число **неархивных** историй с участием человека; обновление при сохранении истории.

### Changed

- **Сборка `.exe` (PyInstaller):** профиль `DelTA.spec` — сборка **onedir** (`COLLECT`), в дистрибутив кладутся `assets` и шаблон `db_template` как `db`; иконка exe: `icon_small.ico` при наличии, иначе `icon.png`, иначе без иконки (чтобы сборка не падала из‑за отсутствующего `.ico`).
- **Пути в frozen-сборке:** `get_app_paths()` и загрузка ассетов учитывают `sys.frozen` / `sys._MEIPASS`.
- **Windows:** иконка в панели задач — `QApplication.setWindowIcon` и явный App User Model ID.
- **Создание/редактирование задачи:** список ответственных не закрывается при клике внутри выпадающего списка; маркированный и нумерованный списки в описании переключаются повторным нажатием; кнопка «Добавить подзадачу» — крупная, только иконка; форма с **вертикальной прокруткой** и меньшей стартовой высотой окна.
- **Статусы историй:** добавлены «Отложен», «Опубликован»; цвет «Рисуется, не дописан» — ярко-красный; миграция существующего `story_statuses.json`.

### Fixed

- **Доска / окно:** при сильном уменьшении окна приложение могло подвисать или вылетать из‑за накопления потоков загрузки фона (`BackgroundImageLoader`); при выходе — предупреждение «QThread: Destroyed while thread is still running». Исправлено: проверка прерывания в потоке, одна активная загрузка с отложенным повтором, квантование размера, дебаунс ресайза, ожидание потоков при закрытии окна; в `BoardPrefetchThread` учтено прерывание.
- **Профиль (админ):** при сужении окна поля и подписи визуально сжимались по вертикали. Форма обёрнута в прокрутку, убраны лишние вертикальные растяжки, зафиксирована минимальная высота строк ввода и политика размера; в стилях для `QLineEdit` задан `min-height`.
- **Истории:** «дрожание» чипов выбора людей заменено на `FlowLayout`; фильтр разделов и синхронизация ролей `for_stories`; выравнивание и цвет заголовка карточки по статусу.
- **Связь задача ↔ история:** при переводе истории в **архив** у всех задач поле `story_id` сбрасывается; при сохранении задачи нельзя указать несуществующее название истории (нужен выбор из списка или прочерк «—»).

## [v1.1.0] - 2026-04-05

### Added

- **Подзадачи (цепочка):** у задачи можно задать упорядоченный список подпунктов с названием и ответственным; на доске — компактная полоса прогресса (серый/зелёный); при количестве подзадач **> 18** на карточке показывается одна линия доли выполнения без точек.
- **Просмотр задачи:** блок «Цепочка подзадач» с многострочной змейкой, подписями и сглаженными переходами между рядами; ограниченная по высоте зона с прокруткой и кнопка **«Развернуть»** для полного просмотра в отдельном окне.
- **Создание/редактирование:** раздел «Подзадачи» со списком строк (название, ответственный, «Готово», удаление).
- **Настройки интерфейса:** параметр «Подзадач в строке» (для детальной цепочки), сохраняется в `ui_settings.json`.
- **Правила данных:** подзадачи нельзя отмечать выполненными с пропуском (только по порядку с начала цепочки); при сохранении проверяется **startline ≤ deadline**; при **создании** новой задачи в поля подставляются **текущие дата/время** и **+1 сутки** для deadline.
- **Панель «Люди» — фильтр по задачам:** в меню фильтра добавлено подменю **«Задачи»** — порядок списка по числу активных задач (как обычно по ролям, сначала с наибольшим или наименьшим числом задач), отбор **«Все»**, **«Нет задач»** (0 задач) и **«С задачами»**; сочетается с фильтрами по ролям и статусам.
- **Описание задачи:** кнопка снятия ссылки с выделенного текста (убирается гиперссылка, подчёркивание и цвет, как у обычного текста в поле).

### Changed

- **Скроллбары:** плоский современный вид, скруглённый ползунок, настраивается через общую таблицу стилей приложения.
- **Просмотр задачи:** у неактивных полей дат и у ответственных скрыты стрелки раскрытия/степпера; календарь у дат отключён; при нажатии «Редактировать» окно просмотра сразу скрывается, открывается только диалог редактирования.

### Fixed

- Время **Startline** при создании задачи: не сбрасывалось в 00:00 из‑за порядка синхронизации дат и `setSelectedDate` у календаря (установка start/deadline под `blockSignals`, без принудительного `setSelectedDate` при инициализации).
